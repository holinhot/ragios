Monitors API

Working with Monitors
POST
To add monitors to the system, they have to be in a json array as the body of a POST request
Example:

POST /monitors/ HTTP/1.0
Content-Type: application/json
 
[{"monitor":"url",
  "every":"5m",
  "test":"1 test feed",
  "url":"http://www.website.com/89843/videos.xml",
  "contact": "obi.akubue@mail.com",
  "via": "gmail",
  "notify_interval": "6h"},
 
 {"monitor":"url",
  "every":"15m",
  "test":"4 test",
  "url":"https://github.com/obi-a/Ragios",
  "contact": "obi.akubue@mail.com",
  "via":"gmail",
  "notify_interval":"3m"}]

This will add 2 monitors to the system one to monitor the url ‘http://www.website.com/89843/videos.xml’ every 5 minutes and the other one to monitor the url ‘https://github.com/obi-a/Ragios’ every 15 minutes. The monitors will start running as soon as they are added.

GET
To get all Monitors in the system

GET /monitors

A monitor in json format will look like this

{"monitor":"url",
  "every":"1m",
  "test":"1 test feed",
  "url":"http://www.website.com/89843/videos.xml",
  "contact": "admin@mail.com",
  "via": "gmail",
  "notify_interval": "6h"}

You can also get monitors using their key/value pair,
Example to get all monitors with contact, admin@mail.com (i.e. key/value pair “contact”: “admin@mail.com”). You can do

 GET /monitors/contact/admin@mail.com

To get all monitors running every minute

 GET /monitors/every/1m

To get all monitors, using the url monitoring plugin

 GET /monitors/monitor/url

Syntax

 GET /monitors/:key/:value

The JSON format for an actively running monitor may look like this:

{"_id":"8bc305cc-e5dd-49dd-a8ea-900dfcc13cd4",
 "_rev":"13-ac82184e494432c2bfa2f5c031441f70",
 "tag":"test",
 "monitor":"url",
 "every":"8m",
 "test":"2 test",
 "url":"https://github.com/obi-a/Ragios",
 "contact":"admin@mail.com",
 "via":"gmail",
 "notify_interval":"3h",
 "creation_date":"November 29, 2011 21:59",
 "state":"active",
 "describe_test_result":"HTTP Request to https://github.com/obi-a/Ragios",
 "time_of_last_test":"November 30, 2011 19:38",
 "num_tests_passed":"12",
 "num_tests_failed":"0",
 "total_num_tests":"12",
 "last_test_result":"200",
 "status":"UP"}

All monitors in the system have a unique _id value. This unique _id is generated by Ragios. To get a monitor by its unique id
Syntax:

 GET /monitors/:id

Example

 GET /monitors/8bc305cc-e5dd-49dd-a8ea-900dfcc13cd4

DELETE
To delete a monitor, we use it’s id with a monitors DELETE request
Syntax

 DELETE /monitors/:id

Example:

 DELETE /monitors/some_monitor_id

This will delete the monitor with the specified id from the system. If the monitor is active, it will be removed from the scheduler and then deleted from the database.

PUT
Edit a monitor
To edit a monitor, let’s say we want to change a monitor’s notification email address, we would use a PUT request;

 PUT /monitors/some_monitor_id HTTP/1.0
 Content-Type: application/json
 
 { "contact" : "skyla@ateam.com"}

The above PUT request will change the monitor’s notification email address to skyla@ateam.com as specified in the body of the request.

To change the monitor’s time interval to run every 18 minutes

 PUT /monitors/some_monitor_id HTTP/1.0
 Content-Type: application/json
 
 { "every" : "18m"}

To change the monitor’s notification email address, time interval and notification method

 PUT /monitors/monitor_id HTTP/1.0
 Content-Type: application/json
 
 { 
   "every" : "1h",
   "contact" : "james@ateam.com",
  }

As specified in the request body, this request will change the monitor to run every hour and notify james@ateam.com if the test fails.

Any value in a monitor key/value pair can be changed like this. Example of a monitor key/value pair in JSON format.

{"monitor":"url",
  "every":"1m",
  "test":"1 test feed",
  "url":"http://www.website.com/89843/videos.xml",
  "contact": "admin@mail.com",
  "via": "gmail",
  "notify_interval": "6h"}

Stop or restart a monitor
To stop a running monitor
syntax:

 PUT /monitors/:id/state/stopped

Example

 PUT /monitors/some_monitor_id/state/stopped

This will stop the monitor with the specified id.

To restart a stopped monitor
syntax:

 PUT /monitors/:id/state/active

Example

 PUT /monitors/some_monitor_id/state/active

This will restart the monitor with the specified id.

When the Ragios server is restarted the monitors in a active state will start running and the stopped monitors will remain stopped, until they are restarted.
